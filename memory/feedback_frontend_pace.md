type: feedback
---

Trabajar el frontend igual que el backend: módulo por módulo, característica por característica, componente por componente. No crear múltiples archivos de golpe.

**Why:** El usuario quiere un proyecto bien hecho y revisado, no código básico generado en bloque.

**How to apply:** Crear solo lo que corresponde al paso actual, mostrar el resultado, esperar que el usuario apruebe o corrija antes de continuar con el siguiente componente o característica.
